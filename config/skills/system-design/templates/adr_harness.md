# Harness Engineering ADR 模板

> 使用方式：LLM 引導使用者填入，呼叫 `ssr(adr)` 記錄。

## ADR-{id}：{標題}

### 🏷 決策重要性

- [ ] 🟥 **重大** — 影響整個系統（資料庫選型、架構模式）
- [ ] 🟨 **中等** — 影響一個模組（library 選擇、API 風格）
- [ ] 🟩 **輕微** — 可跳過 ADR（實作細節、命名慣例）

### 📋 Context（背景）

```
為什麼需要這個決定？
目前的限制是什麼？
有哪些相關的 ADR？
```

### 🤔 Alternatives（考慮過的選項）

```
選項 1: {名稱}
  - 優點：
  - 缺點：
  - 原因（為什麼不選）：

選項 2: {名稱}
  - 優點：
  - 缺點：
  - 原因（為什麼不選）：
```

### ✅ Decision（決定）

```
選了什麼？理由是什麼？
```

```

```
